## Repository

## Stack
- **Frontend**: Next.js 16 App Router + TypeScript + Tailwind CSS v4 + shadcn/ui (base-nova)
- **Backend**: Next.js API routes
- **Database**: Supabase — URL and keys in artifacts/venture.config.json
- **Payments**: Stripe (products already created — keys in artifacts/venture.config.json)
- **Deployment**: Vercel (auto-deploys when you push to main)

## Docs (read in this order)
1. `artifacts/eng_doc.md` — technical spec, schema, API routes, page list
2. `artifacts/prd.md` — product requirements, user stories, core features
3. `artifacts/brand_brief.json` — name, colors, fonts, voice
4. `artifacts/venture.config.json` — product config, pricing tiers, Supabase + Stripe env vars
5. `artifacts/compliance_matrix.json` — PRD feature traceability (if present)

## Quality bar
- Every page from the Engineering Doc must be fully functional
- Mobile-responsive layouts throughout
- No "coming soon" or placeholder pages
- Run `npm run build` and fix all TypeScript errors before finishing

## After building
1. `git add -A && git commit -m "Initial MVP build"`
2. `git push` → Vercel auto-deploys
3. Come back to Venture OS dashboard to connect the live URL
